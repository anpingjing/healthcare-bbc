# 企微健康社群「4R角色」PRD文档（补充版）

## 1. 文档概述
### 1.1 文档目的
本文档定义企微健康社群中**家庭医生、健康管家、权益顾问、专属保险顾问**四大角色的定位、核心职责、服务边界及AI+人工协作实现方案，新增**自动生成个性化健康管理方案**及配套回访、触达、信息收集机制，为产品开发、运营落地提供明确依据。

### 1.2 适用范围
- 产品设计、开发、测试团队
- 运营团队（角色调度、服务质量管控）
- 医疗、保险等合作方

---

## 2. 角色定义与核心职责

### 2.1 家庭医生（Family Doctor）
#### 角色定位
专业医疗核心角色，是用户健康诊疗的第一责任人，提供在线医疗服务与长期健康管理。

#### 核心职责
- **在线问诊**：针对用户常见健康问题提供专业诊疗建议，开具电子处方（合规范围内）。
- **健康指导**：根据用户健康档案，提供慢病管理、康复指导、生活方式调整建议。
- **初步评估**：对用户症状细节、过往病史进行初步问询，评估病情紧急程度。
- **就医引导**：根据评估结果，引导用户至互联网医院问诊、线下医疗机构检查，或在紧急情况下呼叫救护车。
- **定期随访**：对慢病患者或术后用户进行定期健康随访，跟踪恢复情况。
- **方案审核**：审核AI生成的健康管理方案，根据专业判断调整优化，确保方案科学性与可行性。

#### 服务边界
- 不进行线下实体诊疗操作，仅提供线上诊疗建议。
- 不处理超出执业范围的医疗行为，复杂病例需转介至专科医生。
- 紧急情况仅做初步处理与引导，不替代专业急救服务。

#### 实现方案（AI + 人工协作）
| 环节 | AI 辅助 | 人工核心 |
|------|---------|----------|
| 症状初筛 | 通过自然语言交互，初步采集用户症状、病史等信息，生成评估报告 | 执业医生负责最终诊断、治疗方案制定 |
| 病历调取 | 自动调取用户健康档案，辅助医生快速了解病史 | 医生主导复杂病例诊疗过程 |
| 用药建议 | 根据诊断结果，生成标准化用药建议模板 | 医生审核并调整用药方案 |
| 方案生成 | 基于用户健康数据生成初步健康管理方案 | 医生审核、优化方案并确认执行 |

### 2.2 健康管家（Health Butler）
#### 角色定位
AI 智能助手，作为用户与其他角色的调度枢纽，7×24h 响应，提供基础健康服务、角色协调及健康管理方案的落地执行。

#### 核心职责
- **基础咨询**：解答用户常见健康疑问（如日常保健、疫苗接种等）。
- **任务提醒**：根据健康计划，自动推送用药提醒、随访提醒、体检提醒等。
- **角色调度**：根据用户需求，转介至家庭医生、权益顾问或专属保险顾问。
- **入群服务**：用户入群时自动发送欢迎语，引导完成轻量化健康采集表单。
- **数据监测**：跟踪用户健康数据（如血压、血糖），异常时及时预警并转介医生。
- **方案执行**：执行健康管理方案中的回访、触达、信息收集任务，同步数据至健康档案。
- **方案更新**：根据收集到的用户信息，自动更新健康管理方案，形成闭环优化。

#### 服务边界
- 不提供专业医疗诊断和保险规划，仅做信息传递与调度。
- 不处理复杂医疗或金融决策，复杂问题需转介对应专业角色。
- 健康管理方案仅为辅助管理工具，不替代专业医疗诊断。

#### 实现方案（AI 为主 + 人工兜底）
| 环节 | AI 主导 | 人工兜底 |
|------|---------|----------|
| 自然语言交互 | 通过大模型实现 7×24h 智能对话，解答基础问题 | 复杂调度时人工介入处理 |
| 任务自动化 | 自动生成并推送健康任务提醒、回访通知 | 用户情绪激动时人工安抚 |
| 智能调度 | 根据用户问题关键词，自动匹配并转介至对应角色 | 无法判断需求时人工引导 |
| 方案执行 | 自动触发回访、触达任务，收集用户反馈 | 用户拒绝回访时人工介入沟通 |

### 2.3 权益顾问（Rights Consultant）
#### 角色定位
健康权益专家，负责用户健康服务权益的推广、使用指导与价值传递，助力健康管理方案落地。

#### 核心职责
- **权益盘点**：自动查询用户账户内的健康权益（如线上问诊券、药械折扣卡、体检绿通等）。
- **权益引导**：在用户有需求或潜在需求时，引导其使用对应权益，提升服务利用率。
- **活动推送**：当有新的健康权益活动上线时，向感兴趣的用户精准推送通知。
- **反馈收集**：定期回访用户，收集对权益服务的满意度反馈，优化权益配置。
- **入群介绍**：用户入群时，主动推送其可享有的健康服务权益概览。
- **方案赋能**：将健康权益融入健康管理方案，为用户提供可落地的权益使用路径。

#### 服务边界
- 不提供医疗诊断和保险规划，仅聚焦健康服务权益的推广与使用。
- 不进行产品销售诱导，推荐以用户需求为导向，每月主动推荐不超过 2 次。

#### 实现方案（AI 辅助 + 人工深化）
| 环节 | AI 辅助 | 人工深化 |
|------|---------|----------|
| 用户画像分析 | 根据用户健康档案、历史咨询记录，分析潜在权益需求 | 复杂权益咨询时人工详细指导 |
| 精准匹配 | 自动匹配用户权益与需求，生成个性化权益使用建议 | 收集用户反馈后人工整理优化 |
| 活动推送 | 通过标签体系，向目标用户推送新权益活动通知 | 权益使用问题人工答疑 |
| 方案赋能 | 将权益融入健康管理方案模板 | 人工审核权益与方案的匹配度 |

---

### 2.4 专属保险顾问（Insurance Consultant）
#### 角色定位
保险规划专家，按需配置，为用户提供个性化保险方案与长期保障管理，作为健康管理方案的风险保障补充。

#### 核心职责
- **需求评估**：入群后首次服务中，了解用户健康状况、已有保障情况及经济状况。
- **方案定制**：基于保障缺口分析，提供个性化保险规划建议，遵循“以用户需求为导向、保障全面且保费合理”的原则。
- **定期回顾**：每季度对用户保险规划进行回顾，根据健康状况和经济变化调整方案。
- **授权管理**：需经用户同意或授权后才能入群，用户可随时要求其退出。
- **方案协同**：将保险保障建议融入健康管理方案，形成“健康管理+风险保障”的闭环服务。

#### 服务边界
- 不强制推销保险产品，所有建议以用户需求为核心。
- 不提供超出保险范畴的医疗或金融建议，专业医疗问题需转介家庭医生。
- 仅在用户授权的社群中提供服务，不主动跨群打扰用户。

#### 实现方案（AI 测算 + 人工决策）
| 环节 | AI 测算 | 人工决策 |
|------|---------|----------|
| 健康风险分析 | 根据用户健康数据，评估潜在健康风险及保障缺口 | 保险顾问调整优化方案 |
| 方案生成 | 基于风险测算，生成多套保险方案模板 | 顾问向用户讲解并协助决策 |
| 方案协同 | 将保险建议融入健康管理方案模板 | 人工审核保险与健康方案的协同性 |

## 3. 健康管理方案生成与执行机制

### 3.1 方案自动生成逻辑
1.  **数据采集**：AI自动采集用户健康档案（病史、慢病、体检数据）、历史咨询记录、症状反馈、生活习惯等信息。
2.  **风险评估**：通过健康风险模型，评估用户健康风险等级（如低、中、高风险）及重点管理方向（如慢病管理、术后康复、亚健康调理）。
3.  **方案生成**：基于风险评估结果，自动生成个性化健康管理方案，包含：
    - 健康目标（如血压控制、体重管理、慢病稳定）
    - 干预措施（如用药指导、饮食建议、运动计划、体检频次）
    - 权益匹配（如可用的问诊券、体检绿通、药械折扣）
    - 回访与触达计划（定期回访时间、触达方式、信息收集内容）
4.  **方案审核**：家庭医生对AI生成的方案进行审核，调整不合理内容，确认后生效。

### 3.2 方案核心板块：回访、触达与信息收集
| 板块 | 核心内容 | 执行方式 |
|------|---------|----------|
| **定期回访** | - 回访频率：根据健康风险等级设定（低风险每季度1次，中风险每月1次，高风险每周1次）<br>- 回访内容：健康指标监测、方案执行情况、症状变化反馈 | 健康管家（AI）自动触发回访通知，通过企微消息、语音或短信触达用户；用户反馈后自动同步至健康档案 |
| **主动触达** | - 触达场景：健康指标异常预警、方案调整提醒、权益到期提醒、节日健康关怀<br>- 触达方式：企微消息、模板消息、智能语音 | AI根据预设规则自动触发触达，复杂场景由人工介入 |
| **信息收集** | - 收集内容：健康指标（血压、血糖、体重等）、症状变化、生活习惯调整、方案执行反馈<br>- 收集方式：智能表单、语音输入、AI对话交互 | 健康管家（AI）引导用户完成信息收集，自动校验数据有效性，异常数据触发医生审核 |

### 3.3 方案执行流程
1.  **方案生效**：家庭医生审核通过后，健康管理方案正式生效，同步至健康管家系统。
2.  **任务触发**：AI根据方案中的回访、触达计划，自动生成任务队列，按时间节点执行。
3.  **用户交互**：健康管家（AI）与用户交互，完成回访、信息收集，同步数据至健康档案。
4.  **方案更新**：AI根据收集到的用户信息，自动更新健康管理方案（如调整运动计划、优化用药建议），并提交家庭医生审核。
5.  **闭环优化**：方案执行周期结束后，生成执行报告，评估健康目标达成情况，为下一轮方案生成提供数据支撑。

## 4. 角色协作流程（新增健康管理方案环节）
1.  **用户入群**：健康管家（AI）自动发送欢迎语，引导完成健康采集表单，建立初始健康档案。
2.  **需求识别**：健康管家（AI）根据用户对话或健康数据，识别需求类型（医疗、权益、保险、健康管理）。
3.  **方案生成**：
    - 健康管理需求 → AI自动生成初步健康管理方案 → 家庭医生审核优化 → 方案生效。
    - 医疗需求 → 转介家庭医生（AI 初筛 + 人工诊疗）→ 诊疗结果融入健康管理方案。
    - 权益需求 → 转介权益顾问（AI 匹配 + 人工引导）→ 权益使用路径融入方案。
    - 保险需求 → 经用户授权后，转介专属保险顾问（AI 测算 + 人工规划）→ 保险建议融入方案。
4.  **方案执行**：健康管家（AI）执行回访、触达、信息收集任务，同步数据至健康档案，自动更新方案。
5.  **服务闭环**：各角色完成服务后，反馈至健康管家，由健康管家（AI）跟进后续任务（如随访提醒、权益使用提醒），形成健康管理闭环。

## 5. 验收标准（新增）
- 健康管理方案覆盖率：≥ 90%（入群用户均生成个性化方案）。
- 回访完成率：≥ 85%（按方案计划完成回访任务）。
- 信息收集准确率：≥ 95%（收集到的健康数据与实际情况一致）。
- 方案更新频次：每季度至少更新1次，高风险用户每月更新1次。
- 用户对健康管理方案满意度：≥ 4.6/5（基于服务后回访）。

---

## 6. 异常处理流程
1.  **用户拒绝回访/触达**：AI记录拒绝原因，自动降低后续触达频率；若连续3次拒绝，转人工客服介入沟通，了解用户顾虑并调整方案。
2.  **信息收集失败**：AI自动重试2次，若仍失败，标记为异常数据，触发家庭医生审核，必要时由人工跟进收集。
3.  **健康指标异常预警**：AI实时监测健康数据，异常时立即触发家庭医生审核，医生确认后通过紧急触达方式联系用户，引导就医或调整方案。
4.  **方案执行偏离**：若用户未按方案执行干预措施（如未按时用药、未完成运动计划），AI自动发送提醒，连续3次未执行转家庭医生介入，调整方案可行性。
